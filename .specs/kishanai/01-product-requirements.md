# KisanAI Product Requirements Document

## Problem Statement
Indian farmers face fragmented, complex, non-vernacular agricultural advisory. Critical information for daily farming decisions is scattered across multiple sources, locked behind language barriers, and often unreliable.

## Target Users
1. Smallholder farmers (primary) — low literacy, smartphone users
2. Progressive farmers — tech-comfortable, seeking better information
3. FPO managers — serving farmer groups
4. NGO field workers — running agriculture programs

## Core MVP Wedge
One trusted assistant for recurring daily farming decisions: disease triage, weather-to-action, mandi price context, and scheme eligibility — in the farmer's language.

## Features (Priority Order)
1. **Chat Assistant** — Ask any farming question, get sourced answers
2. **Disease Detection** — Upload crop photo, get diagnosis with confidence
3. **Weather Advisory** — Location-based weather with farming actions
4. **Mandi Prices** — Current market prices with trends
5. **Government Schemes** — Eligibility checker with plain-language explanations

## What to Build Now (MVP)
- Landing page with pitch content
- Farmer app with all 5 modules
- AI service layer with mock mode
- API routes with validation
- Compliance pages
- Admin dashboard scaffold

## What to Defer
- Real AI integration (use mock for demo)
- Database integration
- User authentication
- WhatsApp integration
- Voice interface
- Offline mode

## Success Metrics
- Farmers onboarded
- Questions asked per user
- Repeat usage rate (7-day)
- Disease triage completion rate
- Farmer satisfaction score
